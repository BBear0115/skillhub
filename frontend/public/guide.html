SkillHub 使用指南

SkillHub 将团队 Skill 管理拆成“候选版本导入、超级管理员审核、团队开放配置”三层职责。

1. 基本流程

  1. 注册账号后，系统会自动创建个人空间。
  2. 团队成员可以把 ZIP 上传到团队空间，形成候选 SkillVersion。
  3. 超级管理员审核版本，决定是否允许进入 MCP。
  4. 团队管理员从已审核 Skill 中选择哪些对本团队 MCP 开放。

2. 角色职责

  • 超级管理员：启动审核工作台、下载 ZIP、查看部署信息、批准、拒绝、清空当前审核通过版本。
  • 团队管理员：管理团队成员、审批加入请求、配置团队开放的已审核 Skill。
  • 普通成员：上传 ZIP、下载 ZIP、查看 Skill 版本历史。

3. 审核工作台

点击“开始审核”后，后端会为该版本准备 review workbench，并提供以下能力：

  • 下载原始 ZIP 快照
  • 查看解压后的部署目录
  • 查看 handler、manifest、tool 列表等部署元信息

4. ZIP 导入规则

  • ZIP manifest 必须包含 name。
  • ZIP manifest 必须包含 version。
  • 导入只会产生候选版本，不会自动进入 MCP。

5. MCP 可见规则

1. 调用者对 workspace 有访问权
2. Skill 存在 current_approved_version_id
3. 团队空间 exposure 已启用
4. MCP 读取的是当前审核通过版本快照

控制台只展示 MCP 入口信息，不再提供前端手动调试与连通测试面板。

6. MCP 入口

单 Skill 入口：

/mcp/{workspace_id}/{skill_id}

工作区聚合入口：

/mcp/workspaces/{workspace_id}

7. 示例 Skill

仓库中的 examples/server-transfer-skill 提供了三个可审核、可部署的示例工具：

  • stream_audio_to_server
  • stream_text_to_server
  • delete_server_streams
