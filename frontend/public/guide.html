SkillHub Guide

SkillHub is an open-source skill management and MCP gateway for individuals, teams, and agents.

1. Basic Flow

  1. Register an account and receive a personal workspace automatically.
  2. Upload ZIP skill packages in your personal space.
  3. Create a team or request access to an existing team for collaboration.
  4. Team admins decide which skills are shared, and members decide which shared skills they want to use.

2. Team Collaboration Model

  • Team admins create, upload, import, and delete team skills.
  • Team members discover and call only the skills exposed to them.
  • Each member can keep a smaller personal selection from the available team skills.
  • Each workspace can issue an API key for agent integration.

3. Two MCP Entry Modes

Single skill endpoint

Use this only for direct troubleshooting or skill-level isolation:

/mcp/{workspace_id}/{skill_id}

Workspace aggregator endpoint

This is the recommended agent entry point. It exposes the real callable tools for the current workspace and keeps execution on the backend server:

/mcp/workspaces/{workspace_id}

Recommended flow:

{
  "jsonrpc": "2.0",
  "id": 1,
  "method": "initialize",
  "params": {}
}

Then call tools/list to get the flattened tool catalog, resources/list and resources/read to fetch SKILL.md instructions, and tools/call to execute the selected tool. Scripts always run on the backend server.

4. Ways to Add Skills

  • ZIP skills for packaged skill implementations.
  • Repository skill collections for archives that include skills-index.json or marketplace.json.

5. Authentication

SkillHub supports two credentials:

  • Bearer token from user login
  • Workspace API key
Authorization: Bearer <token-or-api-key>
